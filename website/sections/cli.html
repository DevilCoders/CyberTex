Command Line

Ship reliable automations from a single ergonomic interface

The SAPL CLI wraps Python tooling with workflow primitives, rich introspection, and a local sandbox. Operators prototype, harden, and publish playbooks without context switching.

Mode aware execution

Switch between inspect, shell, and deploy modes using consistent flags. Structured JSON output powers CI/CD pipelines and observability.

  • Deterministic dry runs for red team approvals
  • Live variable editing with stateful sessions
  • Human friendly summaries alongside machine data

Integrated QA

Built-in linting enforces guardrails around privileged actions. Inline diffs highlight changes to secrets, credentials, and execution flow before deployment. 

$ sapl inspect playbook.yaml --json \ 
  | jq '.checks[] | select(.status != "pass")'
