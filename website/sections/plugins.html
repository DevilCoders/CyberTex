Plugin Ecosystem

Compose bespoke automation by extending the interpreter

Plugins unlock contextual awareness. Register custom builtins that expose enterprise APIs, ticketing flows, or hardware integrations while inheriting SAPL's guardrails.

Flexible packaging

Distribute plugins with Python packages, wheels, or Git repositories. Operators point the CLI at --plugin-dir or configure discovery via pyproject.toml.

  • Namespace isolation
  • Signed artifacts
  • Version pinning

Runtime introspection

Use :state in the interactive shell to inspect loaded plugins, exported commands, and permission models. Telemetry feeds the website for auditability.

Combine human approvals with automated escalation. SAPL records rationale alongside each privileged execution path.

Safety patterns

Author plugins with built-in validation hooks and circuit breakers. Preflight actions to verify environment posture before running expensive or destructive steps.

  • Audit logging for every privileged call
  • Structured retries with backoff
  • Mandatory approvals for sensitive modules
